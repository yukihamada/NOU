🛠 困ったときは

よくある問題と解決策です。

NOU がメニューバーに出てこない
「アプリケーション」フォルダから NOU.app を直接ダブルクリックしてください。初回起動時に Gatekeeper の確認が出ることがあります（「開く」を選択）。それでも出ない場合は、macOS システム環境設定 → プライバシーとセキュリティ で許可してください。
チャットすると「モデルが見つかりません」と出る
ダッシュボード (http://localhost:4001) の「モデル」タブで、お使いの Mac に合うモデルを1つダウンロードしてください。ダウンロードが終わると自動で使えるようになります (数分かかります)。
応答がとても遅い
モデルのサイズが大きすぎる可能性があります。RAM に対して大きすぎると swap が発生して極端に遅くなります。8GB なら 4B、16GB なら 14B、といった目安で選びなおしてください。また他の重いアプリ (Chrome の多タブ、動画編集など) を閉じるのも効果的です。
Claude Code から接続できない
ターミナルで以下を確認してください:
echo $ANTHROPIC_BASE_URL http://localhost:4001 が返ってくれば OK です。そうでなければ ~/.zshrc に export ANTHROPIC_BASE_URL=http://localhost:4001 を追記して source ~/.zshrc してください。また Claude Code の KV キャッシュ最適化のため ~/.claude/settings.json に "CLAUDE_CODE_ATTRIBUTION_HEADER": "0" を設定することで速度が10倍改善します。
モデルのツール呼び出しが失敗する
Gemma4 系はツール呼び出しに既知のバグがあり Claude Code では使えません。代わりに Qwen3.5 14B、Devstral Small 2、GLM-4.7 Flash などをお使いください。また Ollama のコンテキスト長が短すぎるのも原因になります:
export OLLAMA_CONTEXT_LENGTH=65536 を ~/.zshrc に追加してください。install.sh 経由でインストールした場合は自動設定済みです。
NOU をアンインストールしたい
NOU.app をアプリケーションフォルダから削除してください。モデルデータは ~/.ollama/models/ に保存されています。NOU が relay ノードとして登録した認証情報は ~/.nou/ に保存されます。完全に消したい場合はこれらも削除してください。
